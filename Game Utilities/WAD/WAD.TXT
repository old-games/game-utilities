WAD-файл устроен так:
+08 DD смещение начала "директории"
В "директории" вначале расположен файл палитр(14 палитр,различающихся оттенком),
его смещение в WAD'е +0C,затем файлы уровней,затем файлы звука,и затем собст-
венно файлы графики.Вначале идут глобальные объекты,из которых составлен экран
игры,затем собственно объекты игрового поля,разбитые на группы нулевыми "файла-
ми"-указателями начала и конца группы(?_START и ?_END).Группы следующие:спрайты-
группа S,единая;текстуры-группы P и F,составные(P1,P2,...).
Элемент "директории" устроен так:
+0 DD смещение файла в WAD'е
+4 DD длина файла
+8 DB*8 имя файла,не всегда корректное,дополненное нулями.

Файлы графики, содержащиеся в любом WAD-файле, за исключением файла DARKWAR.WAD
из игры RISE OF TRIAD, подразделяются на 5 типов.
Файл первого типа устроен так:
+0 DW ширина картинки
+2 DW высота картинки
+4 DW точка вставки по X
+6 DW точка вставки по Y
+8 DD смещение в файле 1-го СТОЛБЦА
+A DD то же для 2-го
+C ... (так же для всех столбцов)
+? ... (столбцы)
Столбец состоит из блоков.Блок устроен так:
+0 DB смещение начала блока по Y;если FF,то это не блок,а байт-концевик столбца
+1 DB длина тела блока
+2 DB ?
+3 ... (тело блока-столбец точек)
+? DB ?
Признаки файла этого типа - 0<w[+0]<320, 0<w[+2]<200, его размер >=w[+0]*4+8 и
>d[w[+0]*4+4].
Файл второго типа представляет собой простой битмап 320*200, хранящийся по
строкам.
Признак файла этого типа - его размер равен 64000 байт.
Файл третьего типа - текстура для пола. Это простой битмап 64*64, хранящийся по
строкам.
Признак файла этого типа - его размер равен 4096 байт.
Файл четвертого типа - также текстура для пола. Это простой битмап 64*65,
хранящийся по строкам.
Признак файла этого типа - его размер равен 4160 байт.
Файл пятого типа - тоже текстура для пола (текстуры такого типа встречены только
в HEXEN.WAD). Это простой битмап 64*128, хранящийся по строкам.
Признак файла этого типа - его размер равен 8192 байт.

Файлы графики, содержащиеся в указанном выше как исключение файле DARKWAR.WAD,
подразделяются уже на целых 8 типов; если в таком разнообразии форматов и есть
некий смысл, то от меня он определенно ускользнул...
Файлы первого типа устроены так:
+0 DB ширина картинки/4
+1 DB высота картинки
+2 ... (тело картинки)
+? DW 0
Тело картинки представляет собой битмап с чередованием пикселов по 4 слоям,
хранящийся по слоям, а внутри каждого слоя - по строкам; FF - прозрачный цвет.
Признак файла этого типа - его размер равен b[+0]*4*b[+1]+4.
Файлы второго типа устроены так:
+0 DW ширина картинки (всегда 80h)
+2 DW высота картинки (всегда 80h)
+4 DD 0
+8 ... (тело картинки)
Тело картинки представляет собой простой битмап, хранящийся по строкам.
Признак файла этого типа - его размер равен 16392 байт.
Файлы третьего типа устроены так:
+0 DW ширина картинки
+2 DW высота картинки
+4 DD 0
+8 ... (тело картинки)
Тело картинки представляет собой простой битмап, хранящийся по столбцам.
Признак файла этого типа - его размер равен w[+0]*w[+2]+8.
Файл четвертого типа устроен так:
+0 DW размер окна? (сдвиг влево на все число и вверх на его половину)
+2 DW число столбцов в картинке
+4 DW реальная высота картинки (ни на что не влияет)
+6 DW дополнительный сдвиг влево
+8 DW дополнительный сдвиг вверх
+A DW смещение в файле 1-го СТОЛБЦА
+C DW то же для 2-го
+E ... (так же для всех столбцов)
+? ... (столбцы)
Столбец состоит из блоков.Блок устроен так:
+0 DB смещение начала блока по Y;если FF,то это не блок,а байт-концевик столбца
+1 DB длина тела блока
+3 ... (тело блока-столбец точек)
Признаки файла этого типа - 0<w[+0]<320, 0<w[+4]<200, w[+12]=w[+2]*2+10, его
размер >=w[+2]*2+10 и >w[w[+2]*2+8].
Файл пятого типа устроен так:
+0 DW размер окна? (сдвиг влево на все число и вверх на его половину)
+2 DW число столбцов в картинке
+4 DW реальный размер по Y (ни на что не влияет)
+6 DW дополнительный сдвиг влево (всегда 0)
+8 DW дополнительный сдвиг вверх (всегда 0)
+0A DW ? (всегда 15h)
+0C DW смещение в файле 1-го СТОЛБЦА
+0E DW то же для 2-го
+10 ... (так же для всех столбцов)
+? ... (столбцы)
Столбец состоит из блоков.Блок устроен так:
+0 DB смещение начала блока по Y;если FF,то это не блок,а байт-концевик столбца
+1 DB длина тела блока
+3 ... (тело блока-столбец точек)
Тело блока отсутствует, если длина тела блока = 0; также, на месте тела блока
может присутствовать _один_ байт 0FEh, означающий, что участок, соответствующий
этому блоку, следует закрасить прозрачным цветом.
Признаки файла этого типа - 0<w[+0]<320, 0<w[+4]<200, w[+14]=w[+2]*2+12, его
размер >=w[+2]*2+12 и >w[w[+2]*2+10].
Файл шестого типа - текстура. Это простой битмап 64*64, хранящийся по столбцам,
FF - прозрачный цвет.
Признак файла этого типа - его размер равен 4096 байт.
Файл седьмого типа - небо. Это простой битмап 256*200, хранящийся по столбцам.
Признак файла этого типа - его размер равен 51200 байт.
Файл восьмого типа - сжатая картинка. Он устроен так:
+000 DW ширина картинки
+002 DW высота картинки
+004 DB*300h палитра, сдвинутая влево на 2
+304 ... (сжатое тело картинки)
Тело картинки сжато некой разновидностью блочного RLE-сжатия; какой именно,
я не выяснял - таких картинок мало и заметной ценности они не представляют.
